### คำถามสำหรับการทดลอง
1. ในทดลองที่ 2 เกิดอะไรขึ้นกับ Queue?
คำตอบ:
ในทดลองที่ 2 Queue จะเต็มหลังจากที่ Task ส่งข้อมูลครบตามความจุ (Queue length) ที่กำหนดไว้
เมื่อเต็มแล้ว Task ที่พยายามส่งข้อมูลเพิ่มจะเข้าสู่ Blocked state (ถ้ามี timeout)
หรือถ้าไม่มีเวลารอ ระบบจะส่งไม่สำเร็จและคืนค่า pdFALSE ทันที
สรุป:
Queue เต็ม → Task ส่งต่อไม่ได้ จนกว่าจะมี Task อื่นรับข้อมูลออกไป
2. ในทดลองที่ 3 ระบบทำงานเป็นอย่างไร?
คำตอบ:
ในทดลองที่ 3 จะมี Task หลายตัวทำงานร่วมกัน โดยใช้ Queue เป็นตัวกลางในการสื่อสารระหว่าง Task
Task ผู้ส่ง (Producer) จะส่งข้อมูลเข้าคิว และ Task ผู้รับ (Consumer) จะดึงข้อมูลออกจากคิว
ระบบจะทำงานแบบ multitasking พร้อมการซิงโครไนซ์ ทำให้ LED หรือผลลัพธ์ใน Serial Monitor เปลี่ยนตามจังหวะการส่ง-รับ
สรุป:
ระบบแสดงให้เห็นการสื่อสารระหว่าง Task ผ่าน Queue ภายใต้การจัดสรร CPU ของ FreeRTOS
3. Load Balancer แจ้งเตือนเมื่อไหร่?
คำตอบ:
Load Balancer จะส่งสัญญาณแจ้งเตือนเมื่อพบว่า ภาระงานของ CPU หรือ Queue บางส่วนมีการใช้งานเกินระดับที่กำหนด
เช่น Queue ค้างนาน หรือ Task บางตัวใช้เวลาทำงานมากเกินไป
สัญญาณนี้มักใช้เพื่อปรับสมดุลการทำงาน เช่น ลด priority ของบาง Task หรือเพิ่มทรัพยากรให้ Task อื่น
สรุป:
Load Balancer แจ้งเตือนเมื่อระบบไม่สมดุล เช่น CPU ทำงานหนักเกิน หรือ Queue เต็มบ่อย
